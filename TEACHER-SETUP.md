# 교사용 설정 가이드 — 학생 진도 저장 & 확인 켜기

이 앱은 기본적으로 **각 기기(브라우저)에만** 기록이 남습니다.
아래 1회 설정을 하면 **① 학생이 어느 기기서든 반·이름으로 이어하기**,
**② 교사가 구글 스프레드시트에서 누가 얼마나 했는지 확인**이 가능해집니다.

전부 무료이고 서버 관리가 필요 없습니다. (구글 계정만 있으면 됩니다.)

---

## 1단계 — 구글 스프레드시트 만들기
1. [sheets.new](https://sheets.new) 로 새 스프레드시트를 만듭니다.
2. 이름을 예: `자동화설비 학습현황` 으로 저장합니다.
   (시트 안 내용은 비워둬도 됩니다 — 앱이 자동으로 `학생현황`, `응시기록` 탭을 만듭니다.)

## 2단계 — Apps Script 붙여넣기
1. 그 스프레드시트에서 상단 메뉴 **확장 프로그램 → Apps Script** 클릭.
2. 편집기에 있던 기본 코드를 모두 지우고, 이 저장소의 **`apps-script/Code.gs`** 내용을 통째로 붙여넣습니다.
3. 💾 저장(Ctrl+S).

## 3단계 — 웹 앱으로 배포
1. 오른쪽 위 **배포 → 새 배포**.
2. 톱니바퀴(유형 선택) → **웹 앱** 선택.
3. 설정:
   - 설명: 아무거나 (예: v1)
   - **실행 계정: 나**
   - **액세스 권한: 모든 사용자**
4. **배포** → 권한 요청이 뜨면 본인 구글 계정으로 **허용**.
   (“이 앱은 Google에서 확인하지 않았습니다” 경고가 나오면 → **고급 → (안전하지 않음) 이동** → 허용. 본인이 만든 스크립트라 안전합니다.)
5. 나오는 **웹 앱 URL** 을 복사합니다. `https://script.google.com/macros/s/……/exec` 형태입니다.

## 4단계 — 앱에 URL 연결
1. 이 저장소의 **`config.js`** 파일을 엽니다.
2. `syncUrl: ""` 의 따옴표 안에 3단계에서 복사한 `/exec` URL을 붙여넣습니다.
   ```js
   window.AUTOEXAM_CONFIG = {
     syncUrl: "https://script.google.com/macros/s/AKfy..../exec"
   };
   ```
3. 저장하고 GitHub에 반영합니다. (터미널에서 이 폴더에서)
   ```bash
   git add config.js
   git commit -m "학생 진도 저장 연동 URL 설정"
   git push
   ```
4. 1~2분 뒤 사이트에 반영됩니다. 이제 학생이 들어오면 **반·이름 입력 화면**이 먼저 뜨고, 이후 자동 저장됩니다.

---

## 사용 방법
- **학생**: 링크 접속 → 반·이름 입력 → 학습. 집·학교·폰 어디서 다시 들어와도 같은 반·이름이면 오답노트·정답률이 그대로 이어집니다.
- **교사**: 만든 **구글 스프레드시트만 열면** 됩니다.
  - `학생현황` 탭: 학생별 정답률·푼 문항·오답 수·CBT 응시 횟수·마지막 접속 시각.
  - `응시기록` 탭: CBT 제출마다 시각·반·이름·회차·점수가 한 줄씩 쌓임 → 누가 언제 시험을 봤는지 확인.

## 자주 묻는 것
- **URL을 안 넣으면?** 지금처럼 기기별 로컬 저장으로 동작합니다(교사 확인 불가). 문제될 것 없습니다.
- **코드를 고쳐서 다시 배포하면 URL이 바뀌나요?** 같은 배포를 **관리**에서 “새 버전”으로 업데이트하면 URL이 유지됩니다. (새 배포를 또 만들면 URL이 새로 생기니 config.js도 갱신)
- **개인정보?** 학생 이름이 교사 본인 구글 스프레드시트에 저장됩니다. 이름 대신 **반+번호**만 쓰게 하고 싶으면 알려주세요(그 방식으로 바꿔 드립니다).
- **비밀번호는?** 없습니다. 반+이름이 곧 식별자입니다(교실용으로 충분). 더 엄격히 하려면 로그인 방식으로 확장 가능합니다.
